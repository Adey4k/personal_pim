## Про проєкт

Продукт являє собою систему для керування персональною інформацією — «розумну» телефонну книгу. 
За допомогою штучного інтелекту (**Gemini API**) система аналізує повідомлення чи нотатки, отримані користувачем, і автоматично створює картки контактів, видобуваючи з них ім’я, телефон, пошту та інші корисні поля.

**Навчальний заклад:** [ВСП «ППФК НТУ «ХПІ»](http://polytechnic.poltava.ua)  
**Виконавець:** Ладіков Максим Юрійович  
**Керівник роботи:** Бабич Олександр Вікторович

## Основні можливості

- **Інтелектуальне введення:** Автоматичне вилучення імені, телефону, електронної пошти та інших даних з неструктурованого тексту за допомогою нейромереж (Gemini API).
- **Управління контактами:** Створення, редагування, видалення та ручне введення даних у специфічні поля картки.
- **Організація бази:** Сортування контактів за іменем, групою або датою народження. Гнучка категоризація (сім’я, робота, друзі тощо).
- **Пошук:** Швидкий та "розумний" пошук інформації за будь-яким фрагментом текстових полів або заміток.
- **Імпорт та експорт:** Завантаження даних із контактної книги телефону або зовнішніх джерел, а також експорт персональної бази у структуровані файли.
- **Планування:** Прикріплення конкретних завдань або нагадувань до обраних карток контактів.
- **Синхронізація та Offline-робота:** Повноцінний перегляд та редагування бази без підключення до мережі з подальшою хмарною синхронізацією даних у реальному часі.
- **Безпека:** Безпечна авторизація через Google-аккаунт та забезпечення конфіденційності.

## Стек технологій

| Технологія | Опис |
|------------|------|
| **Dart / Flutter** | Мова програмування та сучасний фреймворк для кросплатформної розробки мобільних застосунків |

## Архітектура

Система побудована за клієнт-серверною моделлю з використанням хмарної інфраструктури за концепцією **Backend-as-a-Service (BaaS)**. 

- Використовується подієво-орієнтований підхід.
- Застосовано сучасний архітектурний принцип Offline-First.
- Для передачі інформації між клієнтом та сервером і забезпечення жорсткої типізації застосовується патерн Data Transfer Object (DTO).
