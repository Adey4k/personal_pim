## Завдання до дипломної роботи
Продукт має являти собою систему для керування персональною інформацією – «розумну» телефонну книгу, яка за допомогою ШІ (використати Gemini API) аналізує повідомлення, отримані користувачем, і автоматично створює картки контактів, видобуваючи з них ім’я, телефон, пошту та інші поля. Слід передбачити інші функції, стандартні для систем такого виду, а також можливість ручного вводу даних, зручний і привабливий інтерфейс користувача, можливості з експорту та імпорту даних тощо.

## Основні можливості
* **Інтелектуальне введення:** Автоматичне вилучення імені, телефону, електронної пошти та інших даних з неструктурованого тексту за допомогою Gemini API.
* **Управління контактами:** Створення, редагування, видалення та ручне введення даних у специфічні поля картки.
* **Організація бази:** Сортування контактів за іменем, групою або датою народження, а також їх категоризація за групами (сім’я, робота, друзі тощо).
* **Пошук:** Швидкий пошук інформації за будь-яким фрагментом текстових полів або заміток.
* **Імпорт та експорт:** Можливість завантаження даних із контактної книги телефону або зовнішніх джерел, а також експорт персональної бази у структуровані файли.
* **Планування:** Прикріплення конкретних завдань або нагадувань до обраних карток контактів.
* **Синхронізація та офлайн-робота:** Можливість повноцінного перегляду та редагування бази без підключення до мережі з подальшою хмарною синхронізацією даних у реальному часі.
* **Безпека:** Авторизація через Google-аккаунт та забезпечення конфіденційності через протоколи шифрування.

## Стек технологій
* **Мова програмування:** Dart.
* **Фреймворк:** Flutter (кросплатформна розробка мобільних застосунків).
* **База даних та Backend:** Firebase (Cloud Firestore, Firebase Authentication).
* **Штучний інтелект:** Gemini API (через Dart-пакет `google_generative_ai`).

## Архітектура
Система побудована за клієнт-серверною моделлю з використанням хмарної інфраструктури за концепцією Backend-as-a-Service (BaaS). Використовується подієво-орієнтований підхід та архітектурний принцип offline-first. Для передачі інформації між клієнтом та сервером і забезпечення жорсткої типізації застосовується патерн Data Transfer Object (DTO).
